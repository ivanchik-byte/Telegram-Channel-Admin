# Telegram Channel Admin

Асинхронный конвейер для автоматизированного сбора, AI-рерайтинга и модерации контента из Telegram-каналов доноров с последующей публикацией в целевой канал.

## Описание проекта

Проект представляет собой микросервисную архитектуру на базе Python 3.11+, PostgreSQL и Redis. Система перехватывает новые публикации из заданных каналов-доноров посредством библиотеки Telethon, фильтрует рекламный контент, переписывает текст с помощью нейросети по заданному промпту (через AsyncOpenAI) и отправляет результат модератору в личные сообщения или выделенную группу для проверки.

Возможности модератора:
- Публикация поста в целевой канал в один клик.
- Отклонение поста.
- Ручное редактирование сгенерированного текста перед публикацией.

### Схема потока данных

1. **Parser (Telethon)**: Отслеживает каналы-доноры. Перехватывает новые сообщения и сохраняет оригинальный текст в базу данных. Дубликаты сообщений отсеиваются через атомарный `INSERT ... ON CONFLICT DO NOTHING`. Пост отправляется в очередь задач.
2. **Redis + Arq**: Принимает задачу на обработку поста.
3. **Worker (Arq + OpenAI)**: 
   - Забирает задачу из очереди.
   - Проверяет наличие рекламного контента по заданным стоп-словам.
   - Предотвращает состояние гонки при обработке дубликатов.
   - Отправляет текст к API нейросети.
   - По готовности атомарно обновляет статус поста в базе данных и формирует карточку для модератора.
4. **Bot (aiogram)**: 
   - При подтверждении модератором бот атомарно переводит пост в статус `published` и отправляет текст в целевой канал.

## Пошаговая инструкция по запуску

### Шаг 1. Получение API-ключей Telegram
1. Перейдите на [my.telegram.org](https://my.telegram.org) и авторизуйтесь.
2. Откройте раздел **API development tools**.
3. Создайте новое приложение. Вы получите `API_ID` (цифры) и `API_HASH` (строка). Они понадобятся для работы парсера.
4. В Telegram найдите [@BotFather](https://t.me/BotFather) и создайте нового бота. Получите `TELEGRAM_BOT_TOKEN`.

### Шаг 2. Настройка окружения
1. Скопируйте пример конфигурационного файла:
   ```bash
   cp .env.example .env
   ```
2. Откройте `.env` и заполните его актуальными данными (подробное описание переменных приведено ниже).

### Шаг 3. Инициализация сессии Telethon (Авторизация парсера)
Поскольку парсер работает от имени пользовательского аккаунта, необходимо сгенерировать файл сессии.
```bash
# Запустите Redis и базу данных
docker compose up -d redis db

# Запустите скрипт авторизации в интерактивном режиме
docker compose run --rm parser python src/login.py
```
Введите номер телефона и код подтверждения из Telegram. После успешного входа в директории `data/` появится файл `anon.session`.

### Шаг 4. Боевой запуск (Production Launch)
Запустите весь конвейер микросервисов:
```bash
docker compose up -d --build
```

Убедиться в корректной работе сервисов можно с помощью команд:
```bash
docker compose ps
docker compose logs -f
```

## Переменные окружения (.env.example)

| Переменная | Описание |
|---|---|
| `POSTGRES_DB` | Имя базы данных PostgreSQL (например, `tg_bot_db`) |
| `POSTGRES_USER` | Пользователь базы данных |
| `POSTGRES_PASSWORD` | Пароль базы данных |
| `DATABASE_URL` | Асинхронный URL для подключения (например, `postgresql+asyncpg://user:pass@db:5432/tg_bot_db`) |
| `REDIS_URL` | Строка подключения к Redis (например, `redis://redis:6379/0`) |
| `TELEGRAM_BOT_TOKEN` | Токен модераторского бота, полученный от @BotFather |
| `ADMIN_IDS` | ID администраторов (модераторов) через запятую |
| `TARGET_CHANNEL_ID` | ID вашего целевого канала для публикаций (начинается с `-100...`) |
| `MODERATOR_CHAT_ID` | ID чата или группы для отправки карточек модерации |
| `API_ID` | ID приложения, полученный на my.telegram.org |
| `API_HASH` | HASH приложения, полученный на my.telegram.org |
| `CHANNELS_TO_TRACK` | Юзернеймы или ID каналов-доноров через запятую |
| `AI_API_KEY` | Ключ доступа к OpenAI (или совместимому API) |
| `AI_BASE_URL` | (Опционально) Пользовательский URL для API нейросети |
| `AI_MODEL` | Название используемой модели (например, `gpt-4o-mini`) |
| `AD_KEYWORDS` | Стоп-слова для выявления рекламы через запятую |
| `OPENAI_EXTRA_BODY` | (Опционально) Дополнительные параметры в формате JSON |

## Безопасность
- **Никогда** не публикуйте файл `.env` или `data/anon.session` в открытом доступе. Они уже включены в `.gitignore` для предотвращения случайного коммита.
- Миграции базы данных применяются автоматически при старте контейнера `migrator`. 
- Вся логика валидации команд и защита от неавторизованного доступа инкапсулирована внутри бота — управление разрешено исключительно пользователям из списка `ADMIN_IDS`.
