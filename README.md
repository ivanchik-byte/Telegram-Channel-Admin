# Telegram Channel Admin (AI Moderator)

Асинхронный конвейер для автоматизированного сбора, AI-рерайтинга и модерации контента из Telegram-каналов доноров с последующей публикацией в ваш целевой канал.

## 📌 Описание проекта

Проект представляет собой микросервисную архитектуру на базе Python 3.11+, PostgreSQL и Redis. Он перехватывает новые посты из заданных каналов (через Telethon), отсеивает рекламу, переписывает текст с помощью нейросети по заданному промпту (через AsyncOpenAI), и отправляет результат модератору в личные сообщения или в группу для проверки.

Модератор может:
- Опубликовать пост в целевой канал (одна кнопка)
- Отклонить пост
- Отредактировать сгенерированный текст вручную перед публикацией

### Схема потока данных

1. **Parser (Telethon)**: Слушает каналы-доноры. Перехватывает новые сообщения и сохраняет оригинальный текст в БД. Дубликаты сообщений отсеиваются через атомарный `INSERT ... ON CONFLICT DO NOTHING`. Пост отправляется в очередь.
2. **Redis + Arq**: Принимает задачу на обработку поста.
3. **Worker (Arq + OpenAI)**: 
   - Забирает задачу.
   - Проверяет на рекламу по стоп-словам.
   - Защищается от гонок при генерации дубликатов.
   - Отправляет текст в AI.
   - По готовности атомарно обновляет статус поста в БД и формирует карточку для модератора.
4. **Bot (aiogram)**: 
   - Модератор нажимает "Опубликовать".
   - Бот атомарно переводит пост в статус `published` и отправляет текст в целевой канал.

## 🚀 Пошаговая инструкция по запуску

### Шаг 1. Получение API-ключей Telegram
1. Перейдите на [my.telegram.org](https://my.telegram.org) и авторизуйтесь.
2. Откройте раздел **API development tools**.
3. Создайте новое приложение. Вы получите `API_ID` (цифры) и `API_HASH` (строка). Они понадобятся для парсера.
4. В Telegram найдите [@BotFather](https://t.me/BotFather) и создайте нового бота. Получите `TELEGRAM_BOT_TOKEN`.

### Шаг 2. Настройка окружения
1. Скопируйте пример конфига:
   ```bash
   cp .env.example .env
   ```
2. Откройте `.env` и заполните реальными данными (см. описание переменных ниже).

### Шаг 3. Инициализация сессии Telethon (Авторизация парсера)
Так как парсер работает от имени пользовательского аккаунта, необходимо сгенерировать файл сессии.
```bash
# Поднимите Redis и БД
docker compose up -d redis db

# Запустите скрипт логина в интерактивном режиме
docker compose run --rm parser python src/login.py
```
Введите номер телефона и код подтверждения из Telegram. После успешного входа в папке `data/` появится файл `anon.session`.

### Шаг 4. Боевой запуск (Production Launch)
Запустите весь конвейер микросервисов:
```bash
docker compose up -d --build
```

Убедиться, что всё работает, можно командой:
```bash
docker compose ps
docker compose logs -f
```

## ⚙️ Переменные окружения (.env.example)

| Переменная | Описание |
|---|---|
| `POSTGRES_DB` | Имя базы данных PostgreSQL (например, `tg_bot_db`) |
| `POSTGRES_USER` | Пользователь БД |
| `POSTGRES_PASSWORD` | Пароль БД |
| `DB_URL` | Асинхронный URL для подключения (например, `postgresql+asyncpg://user:pass@db:5432/tg_bot_db`) |
| `REDIS_URL` | Подключение к Redis (например, `redis://redis:6379/0`) |
| `TELEGRAM_BOT_TOKEN` | Токен модераторского бота от @BotFather |
| `ADMIN_IDS` | ID администраторов (модераторов) через запятую |
| `TARGET_CHANNEL_ID` | ID вашего целевого канала для публикаций (начинается с `-100...`) |
| `MODERATOR_CHAT_ID` | ID чата или группы для карточек модерации |
| `API_ID` | Ваш ID приложения с my.telegram.org |
| `API_HASH` | Ваш HASH приложения с my.telegram.org |
| `CHANNELS_TO_TRACK` | Юзернеймы или ID каналов-доноров через запятую |
| `AI_API_KEY` | Ключ доступа к OpenAI (или совместимому API) |
| `AI_BASE_URL` | (Опционально) Кастомный URL для API нейросети |
| `AI_MODEL` | Название модели (например, `gpt-4o-mini`) |
| `AD_KEYWORDS` | Стоп-слова для рекламы через запятую |
| `OPENAI_EXTRA_BODY` | (Опционально) Дополнительные параметры в формате JSON |

## 🛡 Безопасность
- **Никогда** не публикуйте `.env` или файл `data/anon.session`. Они уже жестко добавлены в `.gitignore`.
- Миграции базы данных происходят автоматически при запуске контейнера `migrator`. 
- Вся логика валидации команд и защиты от неавторизованных кликов инкапсулирована внутри бота — управление только у пользователей из списка `ADMIN_IDS`.
