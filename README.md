# Telegram Channel Admin (AI Moderator)

Мы построили конвейер, который автоматизирует рутину администратора Telegram-каналов. Он сам читает доноров, фильтрует мусор, переписывает посты с помощью нейросетей и отдает готовый результат на ревью. Модератору остается только нажать кнопку публикации.

Система решает главную проблему контент-мейкеров. Никто не хочет тратить часы на ручной рерайт чужих новостей. Вместо прямого копирования мы пропускаем каждый пост через LLM, чтобы получить уникальную подачу.

## Как это устроено

Проект разбит на несколько микросервисов. Это нужно для того, чтобы парсинг не тормозил модерацию, а тяжелые запросы к нейросети не блокировали базу данных.

1. **Parser (Telethon)**
Работает как обычный пользователь Telegram и слушает выбранные каналы. Когда появляется новый пост, парсер сохраняет его в PostgreSQL. Мы используем атомарный `INSERT ... ON CONFLICT DO NOTHING`. Из-за этого дубликаты от разных доноров отсеиваются еще на уровне базы данных.

2. **Очередь (Redis + Arq)**
Сразу после сохранения парсер отправляет задачу в очередь. Мы выбрали Arq за простоту и отличную интеграцию с asyncio.

3. **Worker (Arq + AsyncOpenAI)**
Воркер берет пост из очереди и проверяет его на рекламу по словарю стоп-слов. Если текст чистый, он уходит к нейросети.
Запросы к API часто падают по таймауту или упираются в лимиты. Из-за этого мы написали кастомный Exponential Backoff. Сессия базы данных на время генерации текста закрывается, чтобы не держать пул соединений подвешенным. Если оригинал уже отфильтрован или забракован, воркер автоматически отменяет обработку его дубликатов.

4. **Bot (aiogram)**
Бот присылает карточку готового поста в чат модераторов. Текст можно опубликовать, отклонить или отредактировать прямо через интерфейс Telegram (команда `/edit`). Все обновления статусов в базе атомарны через оптимистичные блокировки. Несколько модераторов не смогут случайно опубликовать один и тот же текст дважды.

## Развертывание и настройка

Система собирается через Docker Compose. Установка занимает около десяти минут.

### 1. Подготовка API-ключей
Сначала вам понадобятся ключи разработчика от самого Telegram:
- Зайдите на [my.telegram.org](https://my.telegram.org) и авторизуйтесь.
- В разделе API development tools создайте приложение. 
- Запишите `API_ID` и `API_HASH`. Они нужны, чтобы парсер мог притворяться живым клиентом.

Затем создайте бота для модерации через @BotFather и сохраните его токен.

### 2. Конфигурация
Скопируйте шаблон `.env.example` в `.env`. Обязательно заполните все поля.

| Переменная | Назначение |
|---|---|
| `POSTGRES_DB` | Имя базы данных PostgreSQL |
| `POSTGRES_USER` | Пользователь |
| `POSTGRES_PASSWORD` | Пароль |
| `DATABASE_URL` | Формат: `postgresql+asyncpg://user:pass@db:5432/db_name` |
| `REDIS_URL` | Формат: `redis://redis:6379/0` |
| `TELEGRAM_BOT_TOKEN` | Токен бота от BotFather |
| `ADMIN_IDS` | ID аккаунтов, которым разрешено нажимать кнопки (через запятую) |
| `TARGET_CHANNEL_ID` | ID канала для публикации (обычно начинается с `-100`) |
| `MODERATOR_CHAT_ID` | Чат или группа, куда бот будет присылать карточки |
| `API_ID` | Из первого шага |
| `API_HASH` | Из первого шага |
| `CHANNELS_TO_TRACK` | ID или юзернеймы каналов для парсинга (через запятую) |
| `AI_API_KEY` | Ключ доступа к OpenAI |
| `AI_BASE_URL` | Кастомный URL, если используете прокси или локальную модель |
| `AI_MODEL` | По умолчанию `gpt-4o-mini` |
| `AD_KEYWORDS` | Слова, при которых пост сразу удаляется (через запятую) |
| `OPENAI_EXTRA_BODY` | Опциональный JSON для тонкой настройки API |

### 3. Авторизация парсера
Парсеру нужен файл сессии. Чтобы его получить, запустим базу и Redis, а затем скрипт логина.

```bash
docker compose up -d redis db
docker compose run --rm parser python src/login.py
```
Скрипт попросит ввести номер телефона и код из Telegram. После успешного входа в папке `data/` появится файл `anon.session`.

### 4. Запуск проекта
Теперь можно поднять все сервисы. Мигратор накатит структуру базы автоматически, после чего он завершит работу. Остальные контейнеры останутся висеть в фоне.

```bash
docker compose up -d --build
```
Проверить статус работы можно командой `docker compose ps`. Почитать логи, если что-то не стартует, можно через `docker compose logs -f`.

## Архитектурные решения и безопасность

Мы отказались от промежуточных статусов базы данных, чтобы избежать рассинхрона между воркером и базой. Пост находится либо в очереди, либо в обработке, либо готов к модерации.

Бот инкапсулирует всю проверку прав. Если кто-то попытается нажать кнопку на карточке, не будучи в списке `ADMIN_IDS`, бот просто ответит ошибкой доступа.

Следите за файлами `.env` и `anon.session`. В публичном репозитории они скрыты через `.gitignore`. Не копируйте их в открытые пространства, иначе вы скомпрометируете свой аккаунт Telegram.
