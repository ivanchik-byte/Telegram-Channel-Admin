# Команды управления конвейером (Docker Compose)

Этот файл содержит все необходимые команды для администрирования бота, воркера, парсера и базы данных.

> Все команды нужно запускать, находясь в корневой папке проекта (где лежит файл `docker-compose.yml`).

## 🚀 Основные команды запуска

**1. Полный перезапуск с пересборкой (если вы меняли код)**
Рекомендуется использовать эту команду после обновления файлов `.py` или установки новых зависимостей:
```bash
docker compose down
docker compose up -d --build
```

**2. Быстрый запуск (если код не менялся)**
Используется для обычного старта системы:
```bash
docker compose up -d
```

**3. Авторизация парсера (разовая операция)**
Генерация файла `anon.session` для работы Telethon (требует ввода телефона и кода из Telegram):
```bash
docker compose up -d redis db
docker compose run --rm parser python src/login.py
```

## 🔍 Мониторинг и логирование

**1. Проверка статуса всех контейнеров**
Позволяет увидеть, какие сервисы запущены и работают без ошибок:
```bash
docker compose ps
```

**2. Чтение общих логов в реальном времени**
Показывает вывод сразу от всех микросервисов. Выход — комбинация `Ctrl+C`:
```bash
docker compose logs -f
```

**3. Чтение логов конкретного микросервиса**
Если нужно изолировать вывод только одного компонента:
```bash
docker compose logs -f bot      # Логи бота модерации (интерфейс, ошибки доступа)
docker compose logs -f worker   # Логи ИИ-рерайтера (вызовы OpenAI, стоп-слова)
docker compose logs -f parser   # Логи парсера (захват новых сообщений из Telegram)
docker compose logs -f db       # Логи базы данных PostgreSQL
```

## 🛑 Остановка и сброс

**1. Безопасная остановка системы**
Останавливает все процессы, но сохраняет базу данных и очередь нетронутыми:
```bash
docker compose down
```

**2. Полный сброс (Hard Reset) ⚠️**
Полностью останавливает контейнеры и **УДАЛЯЕТ** базу данных и кэш Redis.
*Используйте осторожно! Вы потеряете историю обработанных постов. Файл `anon.session` при этом сохранится.*
```bash
docker compose down -v
```
